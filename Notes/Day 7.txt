Introduction to .NET & Environment
	What is .NET?
		.NET is a modern, open-source, cross-platform development framework developed by Microsoft. It supports building applications for:
			Web(ASP.NET Core)
			Desktop(Windows Forms, WPF)
			Cloud & Microservices
			Mobile(MAUI/Xamarin)
			Gaming(Unity)
		.NET provides a runtime + libraries + tools to simplify development.
	Core components of .NET
		Core CLR(Core Common Language Runtime)
			Assist in Code Execution
			Handle:
				Memory Management (GC)
				Security
				Exception Handling
		CIL (Common Intermediate Language)
			.NET Language code is compiled into IL, not the machine code directly.
			CLR convert IL -> machine code at runtime.
		CTS(Common Type System)
			Defines how data types are declared & used.
		CLS(Common Language Specification
			Set of rules allowing multiple languages to work together.
		BCL(Base Class Library)
			Provides ready-made classes like:
				System.Console
				System.Collections.ArrayList

C# Language Fundamentals
		• Error handling & logging patterns
			What is Error handling?
				Error handling ensures your spplication doesn't crash unexpectedly & can handle failures gracefully.
				Best practices for Exception Handling:
					Catch specific exceptions first
					Don't ever swallo exception silently
					Use finally or using for cleanup
					Avoid using exceptions for normal logic
					Log errors properly.
			Logging Patterns in .NET
				Why logging?
					It helps you to:
						debug issues.
						monitor applications.
						track errors in production.
				How to implement logging in .NET?
					Built-in Logging(ILogger)
				Logging levels
					Trace
					Debug
					Information
					Warning
					Error
					Critical
				Popular logging libraries
					Serilog
					NLog
					Log4Net
					Azure App Insights
			Logging Best Practices
				Use structured logging
				Avoid logging sensitive data
				Use appropriate levels
				Log exceptions with stack trace
					
		• Testing approaches overview
			Why Testing?
				It ensures:
					correctness of the code.
					Stability
					maintainability
			Types:
				Unit Testing - Tests small units
				Integration Testing - Tests interaction between components
				Functional Testing(End-To-End Testing) - Tests full application flow
			Best Practices for testing
				Follow AAA pattern
					Arrange
					Act
					Assert
				Write small, independent tests
				Use mocking for isolating dependencies.
