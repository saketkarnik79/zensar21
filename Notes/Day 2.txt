Quality guardrails
	Avoid "Magic Code"
		Bad Code Example: 
			if(amount > 1000)
		Good Code Example:
			private const decimal MaxOrderAmount = 1000;
			if(amount > MaxOrderAmount)
	Validate Inputs Explicitly
		if(string.IsNullOrWhitespace(dto.CustomerName))
			throw new ArgumentException("Customer name required.");
	Review EF Queries
		Watch for:
			Missing AsNoTracking()
			N + 1 queries
			Unfiltered queries
	Ensure proper error handling always
	Avoid blind Copilot acceptance
	
Unit Testing, Integration Testing & Test Coverage
	• Anti-patterns to avoid
		Blind Test Generation
			Generate 100 Tests (Bad)
		Testing implementation Instead of Behavior
			Assert.Equal(1, repo.CallCount);(Bad)
			Assert.Equal("Test", result.CustomerName);
		Always passing tests (False positives)
			Assert.True(true);
		Over mocking
			Leads to fake systems
			Breaks real-world behavior
		Ignoring failure paths
			Always test:
				Exceptions
				Invalid inputs
				External failures
