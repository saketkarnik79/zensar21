What is Gen AI?
	AI systems designed to generate novel content rather than just analyze or classify data.
Core mechanism used by Gen AI:
	It uses advanced models like transformers, GANs(Generative Advarsarial Networks), diffusion models, variational auto encoders to learn from the training data & produce new outputs.
Examples Gen AI
	Text: Genereate new text using LLMs. This text could be essays, emails, poetry, story, etc.
	Images: Artistic images, realistic images based on text prompts.
	Code: Generating code snippets from text prompts.
	Music & Video:
How it works?
	Training: Models are trained on massive datasets (text, images, audio, video, etc.)
	Pattern recognition: The AI learns statistical patterns & relationships.
	Generation: When propted, it produces new outputs that mimic the style, structure, or logic of the training data.

Prompt Engineering for Developers
	• Prompt fundamentals — what to include (intent, constraints, patterns) and what to avoid
		What to include in a Good Prompt?
			A high-quality prompt must contain:
				Intent
					e.g. Create an ASP.NET Core Api endpoint to retrieve customer orders
				Constraints
					Specify requirements:
						Frameworks
						Coding Standards
						Performance/security needs
					e.g. Use async/await , Entity Framework Core and return IActionResult with proper status codes
				Patterns
					e.g. Follow repository pattern and use dependency injection
		What to avoid?
			Vague prompts
			Overloaded prompts
			Missing constraints
			Sensitive data			
	• Inline prompts vs chat-based prompts
		Inline prompts (comments in code)
			Best for:
				Code generation
				Small scoped tasks
					e.g. // Create a service method to calculate invoice total with tax & discounts
				Fast & contextual
				Uses existing file structure
				Limited reasoning depth
		Chat Based Prompts
			Best for:
				Design dicussions
				Debugging
				Refactoring
					e.g. Refactor this service to follow SOLID principles & improve testability
				Support iteration
				Handles complex instructions
				Requires explicit context
		When to use What?
		
		Scenario				Use
		Generate API method		Inline
		Fix performance issue	Chat
		Explain legacy code		Chat
		Add DTO mapping			Inline
		
	• Refinement techniques — iterative prompting & correction prompts
		Iterative prompting
			Start simple -> refine progressively
			e.g. 
				Generate a repository method to fetch orders
				Add pagination & filtering by status
				Optimize query using EF Core best practices
		Correction prompts
			Fix null handling and add proper exception management
			Rewrite this using async/await instead of synchronous calls
		Style enforcement prompts
			Follow clean architecture principles and use dependency injection
		
	• Using comments as design intent
		Comments act as high-signal prompts for Copilot.
		e.g.
			// Domain service to process order checkout
			// Steps:
			// 1. Validate Order
			// 2. Calculate Total
			// 3. Apply Discounts
			// 4. Save Order
			// Reject order if total is zero or negative
	• Role-based prompts (e.g., Act as a senior .NET architect...)
		It improves output quality by guiding tone & expertise:
			Architect level
				e.g. Act as a senior .NET architect. Design a scalable order processing service using clean architecture.
			Security-focused
				e.g. Act as a security expert. Review this code for OWASP vulnerabilities.
			Performance-focused
				e.g. Act as a performance engineer. Optimize this EF Core query.
				
	• .NET-specific prompt patterns — API controllers, domain services, repositories, DTO mappings, validation logic
		
	e.g.
		Act as a senior .NET developer.

		Create an ASP.NET Core API endpoint to create an order.
		Constraints:
		- Use async/await
		- Validate input using FluentValidation
		- Use EF Core repository pattern
		- Return appropriate HTTP status codes
		- Handle exceptions properly

		Include DTO mapping and dependency injection.
